## Plan — Add OTP gate to the home page TestDrive form

**Goal:** The home page "Book a Test Drive" section gates behind phone OTP, exactly like the contact, service, and wizard forms. Verified once → unlocked everywhere (the universal `VerifiedPhoneProvider` from the prior change handles this). Submissions still land in the Google Sheet via `submitLead`.

### Context confirmed
- The three existing OTP forms (`ContactUs`, `ServiceBooking`, `TestDriveWizard`) all use the same pattern: `<OtpGate>` wraps an `Inner` component that receives `verifiedPhone`, assigns `const mobile = verifiedPhone`, and passes it to `submitLead` as `mobile_number`.
- `TestDrive.tsx` is the only holdout — it renders its own `Mobile Number` input and reads `form.mobile`.
- `submitLead("test_drive", {...})` already works for this component. Adding OTP doesn't touch the endpoint — it only changes the source of `mobile_number`.

### File to change: `components/TestDrive.tsx` (only edit)

1. **Split into `TestDriveInner` + `TestDrive`**, mirroring the other forms:
   - `TestDriveInner({ verifiedPhone, requestChangePhone })` — receives the verified phone
   - `export default function TestDrive()` — wraps `<OtpGate>` around `<TestDriveInner>`

2. **Replace the mobile `<input>` field with the verified-phone chip** — same markup used in `ContactUs`/`ServiceBooking` (blue-tinted card showing "+91 {mobile}" + "Verified" badge, clickable to trigger `requestChangePhone`).

3. **Wire verified phone to submission:**
   - Add `const mobile = verifiedPhone;` where the other state is declared
   - Remove `mobile` from the `form` state object and from `validate()`
   - In the `submitLead("test_drive", {...})` call: `mobile_number: form.mobile.trim()` → `mobile_number: mobile`

4. **Imports:**
   - Add `import { OtpGate } from "./OtpGate";`
   - Add `Phone` to the icons import

### Not touching
- Form layout, image panel, success state, agreement text
- `submitLead`, the endpoint, `NEXT_PUBLIC_SHEET_ENDPOINT`, or any sheet logic
- The other three OTP forms — already work with the universal provider
- `lib/submitLead.ts`, `VerifiedPhoneProvider`, `OtpGate` — unchanged
- Field labels, validation messages, styling — kept identical except for the mobile chip

### Edge cases
- **Verified state still gated by OtpGate** — the form grid only renders after OTP success. The `requestChangePhone` chip gives users a path back to re-verify.
- **Submission integrity** — `mobile_number` is now always a verified 10-digit number (OTP-enforced), so the sheet receives cleaner data than the old free-text input.
- **Universal verification** — if the user already verified elsewhere, this form opens directly to the car/location fields, no re-OTP. That's the intended behavior from the prior change.
