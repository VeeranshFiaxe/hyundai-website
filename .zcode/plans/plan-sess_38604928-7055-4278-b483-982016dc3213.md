## Goal
Replace every external `https://` image (and PDF) reference with locally-served files under `public/`, so the repo is self-contained and pushable to GitHub with zero runtime dependence on hyundai.com / aeplcdn / unsplash CDNs.

Per your decisions: **download ALL ~3,300 360° frames**, **download the 13 brochure PDFs**, **leave the 1 Promise video remote** (streamed, too large).

**Includes the full per-model car detail gallery** — all ~138 `officialAsset()` images across 13 models in `lib/car-details.ts` `modelFeatureGallery` (dashboard, seats, sunroof, engine, exterior/interior close-ups, etc.). These are the images that appear in the "Official image gallery" section on each `/cars/[slug]` page.


## Step 1 — write `scripts/fetch-car-images.mjs`
Mirrors the existing `scripts/fetch-location-images.mjs` style (Node ESM, `fetch`, progress logging). It will:
1. Hard-code the full URL→dest map (derived mechanically from the current `official`, `officialShot`, `banner`, `findACar`, `hy`, `stock` helpers + the `brochurePathBySlug` table + colour lists + **the full `modelFeatureGallery` array**).
2. Download every file with retries + a short concurrency limit (e.g. 8) so ~3,560 files finish in reasonable time.
3. For 360 frames, iterate `modelFolder × colorSlug × frame 0..35`.
4. For gallery images, iterate all 13 slugs × each `src` in `modelFeatureGallery`.
5. Skip files that already exist on disk (re-runs are cheap and resumable).
6. Write `lib/image-manifest.ts` at the end from the same map.
7. Log any 404s so we can spot dead URLs.

Committed to the repo (matches existing convention); the documented way to refresh assets.

## Step 2 — rewrite the code references
After the manifest exists, the code changes are mechanical and low-risk:

- **`lib/data.ts`**: replace `official()`, `officialShot`, `banner()`, `findACar()`, `hy()`, `stock()` call sites with manifest lookups. Remove the now-unused helpers. `testDriveImage`, `serviceHeroImage`, `aboutHeroImage`, `aboutCultureImage`, `blogHeroImage`, testimonial avatars, `heroSlides[].image`, `cars[].image`, `cars[].colors[].image` all become `/...` local paths.
- **`lib/car-details.ts`**: every `officialAsset()` in `modelFeatureGallery` reads from `galleryBySlug[slug]` (preserving label + alt + order exactly); `getCarBrochure()` returns `/brochures/<slug>.pdf`; `getCarGallery()` 360-frame builder points at `/cars/360/...`.
- **`components/Car360Viewer.tsx`**: change `BASE_URL` to `/cars/360` and update `frameUrl()` to `/cars/360/${modelFolder}/${colorSlug}/${colorSlug}_${frame}.png`. No behavioural change.
- **`components/HyundaiPromise.tsx`**: `PROMISE_HERO_IMAGE` → `/campaigns/promise-hero.jpg`.
- **`components/PromiseExperience.tsx`**: video URL **stays** `https://clicktobuy...` (per your decision). Add a comment noting it's intentionally remote.
- **`app/contact-us/page.tsx`** & **`app/locate-service-centre/page.tsx`**: use manifest for `aboutHeroImage`/`serviceHeroImage`.
- **`app/about/page.tsx`**: already local — no change (its locally-defined consts shadow the exports).
- **`next.config.ts`**: drop `imgd.aeplcdn.com`, `images.unsplash.com`, `www.hyundai.com` from `remotePatterns`. Keep `bunny-wp-pullzone...` (still a fetch source for locations).
