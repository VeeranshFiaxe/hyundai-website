## Goal
Turn the home-page "Latest from our Blog" section into a real blogs destination: a dedicated `/blogs` listing page + individual `/blogs/[slug]` article pages, with the header (and footer) "Blogs" buttons now routing there. Expand the data from 4 to ~8 posts so the category filter and grid feel full.

All new code follows existing repo conventions (App Router server pages with `metadata` + JSON-LD, a client grid component mirroring `CarsGrid`, content driven from `lib/data.ts`). New blog images are already verified returning 200 on the `imgd.aeplcdn.com` CDN (which is already in `next.config.ts` remotePatterns).

---

## 1. `lib/data.ts` - data model + nav (edit)
- **Extend `Blog` type** with article fields:
  ```ts
  type Blog = {
    slug: string;          // new – route key
    date: string;
    category: string;
    title: string;
    excerpt: string;       // new – card + article dek
    readingTime: string;   // new – e.g. "4 min read"
    image: string;
    alt: string;
    content: { heading?: string; body: string[] }[]; // new – structured body
  };
  ```
- **Add 4 verified image paths** to the `shot` object (Exter exterior/rear/interior, IONIQ 5), used via the existing `hy(shot.x, 900, 600)` helper.
- **Enrich the existing 4 posts** (add `slug`, `excerpt`, `readingTime`, `content`) and **add 4 new posts** (Exter buying guide, first-time buyer, IONIQ 5 EV, service-intervals guide) so categories span Models / Ownership / Service / Finance / Electric. Dates stay late-May/June 2026. Article body = draft marketing copy (3–4 sections each) the owner can edit.
- **Change nav link** `{ label: "Blogs", href: "/#blogs" }` → `"/blogs"`. (Navbar's `routeOf`/active logic then correctly highlights Blogs on `/blogs` and `/blogs/[slug]` via `startsWith(route + "/")`, and it no longer competes with Home for `route === "/"`.)

## 2. `app/blogs/page.tsx` - listing page (new, server component)
Mirrors `app/cars/page.tsx`:
- `metadata` (title, description, canonical `/blogs`, OG, twitter).
- JSON-LD `@graph`: `Blog` + `BreadcrumbList` + `ItemList` of `BlogPosting` entries (same inline-`<script>` + `.replace(/</g, "\\u003c")` pattern as cars/about).
- Hero header (eyebrow, h1, subtitle) in `bg-bg-2`.
- `<BlogGrid />` client component.
- Closing brand CTA band.

## 3. `components/BlogGrid.tsx` - filterable grid (new, client component)
Mirrors `CarsGrid.tsx`:
- Category filter buttons derived dynamically from the blog categories (`"All"` + unique categories).
- Grid of `<BlogCard />`, each linking to `/blogs/{slug}`.

## 4. `components/BlogCard.tsx` - shared card (new)
Single card (image, category, date, title, excerpt, "Read more") as a `<Link href="/blogs/{slug}">`. Reused by `BlogGrid` and the article page's related-posts section so styles don't drift.

## 5. `app/blogs/[slug]/page.tsx` - article page (new, server component)
Per the Next.js 16 docs (read in `node_modules/next/dist/docs/`): `params` is a `Promise` here, so `await params`; include `generateStaticParams()` + `generateMetadata({ params })`.
- Unknown slug → `notFound()`.
- Article layout: hero image, category badge, h1, date · readingTime, excerpt, then structured body sections (headings + paragraphs) styled manually with Tailwind (no typography plugin exists).
- JSON-LD `BlogPosting` (headline, datePublished, image, articleBody, author/publisher) + `BreadcrumbList`.
- "Related posts" section (same category preferred, else others) using `BlogCard`.

## 6. `app/blogs/[slug]/loading.tsx` - loading skeleton (new)
Recommended by the linking/navigating docs for dynamic routes (enables streaming/partial prefetch). Simple skeleton card.

## 7. `components/Blogs.tsx` - home section wired to the new page (edit)
- "View All Blogs" → `/blogs` (was `#blogs` anchor).
- Each card "Read more" → `/blogs/{slug}` (was `#blogs`).
- Slice to `blogs.slice(0, 4)` so the home section stays a 4-up teaser; the full set lives on `/blogs`.

## 8. `components/Footer.tsx` - footer link (edit)
Blogs quickLink `/#blogs` → `/blogs` (same destination as header, avoids inconsistency).

## 9. `app/sitemap.ts` - index new routes (edit)
Add `/blogs` (priority 0.7) and each `/blogs/{slug}` (priority 0.6), matching the existing entries' shape.
